## 2024-04-01 - Prevent OS-wide re-renders via Zustand object reference stability
**Learning:** Zustand's `useStore()` hook subscribes to the *entire* state object by default. If array mappers like `.map()` blindly return new objects (`{...p, status: 'background'}`) even when data hasn't changed, it breaks referential equality for all items. Combined with full state subscription, this causes massive cascading re-renders across the application (like re-rendering every background `AppWindow`).
**Action:** Always use targeted selectors (e.g., `useOSStore(state => state.backgroundApp)`) instead of destructuring the whole store. In reducers/actions, check if the data changed (`p.status === 'background'`) and return the original object `p` if unchanged to preserve referential equality for `React.memo`.
